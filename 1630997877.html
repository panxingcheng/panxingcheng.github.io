      # 在 Docker 容器中部署 Logstash

      • 拉取 Logstash 镜像

        $ docker pull docker.elastic.co/logstash/logstash:7.16.1
        1

      • Pipeline 配置

        将管道配置放在 Logstash 能够找到的地方是非常重要的。默认情况下，容器将在 /usr/share/logstash/pipeline/ 中查找管道配置文件。

        在这个例子中，我们使用一个绑定挂载的卷通过 docker run 命令来提供配置：

        $ docker run -it -v ~/pipeline/:/usr/share/logstash/pipeline/ docker.elastic.co/logstash/logstash:7.16.1
        1

        Logstash 将主机目录 ~/pipeline/ 中的每个文件解析为管道配置。

      • 设置

        通过绑定挂载提供设置文件。Logstash 希望在 /usr/share/logstash/config/ 找到它们。

        可以提供一个包含所有所需文件的完整目录：

        $ docker run -it -v ~/settings/:/usr/share/logstash/config/ docker.elastic.co/logstash/logstash:7.16.1
        1

        或者，单个文件也可以挂载：

        $ docker run -it -v ~/settings/logstash.yml:/usr/share/logstash/config/logstash.yml docker.elastic.co/logstash/logstash:7.16.1
        1
        绑定挂载的配置文件将在容器中保留它们在主机系统上拥有的相同权限和所有权。确保设置权限，使文件能够被容器的日志存储用户（UID 1000）读取，理想情况下，文件不能被写入。
        1
